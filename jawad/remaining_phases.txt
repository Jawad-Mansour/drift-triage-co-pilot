SYSTEM MENTAL MODEL
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  AGENT (brain)    →  QUEUE (storage)  →  WORKER (messenger)  →  PLATFORM (hands)
  decides severity     Redis FIFO          delivers with retry      executes ML work
  decides action       persisted           5x exp backoff           trains XGBoost
  asks user (HIL)      across restarts     DLQ on failure           serves predictions
                                                                     promotes models

Component         Talks to user?   Role
─────────────────────────────────────────────────────────────
Agent             YES (HIL inbox)  Brain — decides, pauses for human approval
Worker            NO               Messenger — carries tasks from queue to platform
Platform          NO               Hands — executes ML operations when told to
Dashboard         YES (via agent)  UI — displays everything, approve/reject buttons
MLflow            NO               Storage — platform reads/writes models to it

COMPLETE END-TO-END FLOW (all phases combined)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1. Platform serves predictions via GET /predict
2. Platform logs each prediction to DB, computes PSI/chi² every N predictions
3. If PSI > threshold → Platform fires POST agent:8002/webhook  ← closes automated loop
4. Agent receives webhook → triage → action → HIL (if needed) → comms
5. User approves HIL → task pushed to Redis queue
6. Worker picks up task (BLPOP) → calls Platform:
     RETRAIN   → POST platform/retrain         → new XGBoost model in MLflow Staging
     ROLLBACK  → POST platform/registry/promote → swap champion back to previous
     REPLAY    → POST platform/replay-test       → re-score test set, log AUC
7. After retrain: Worker calls POST agent/investigations/{id}/notify_retrain_complete
8. Agent creates PROMOTE_TO_PRODUCTION HIL (2nd human approval)
9. User approves → Agent calls POST platform/registry/promote
10. Platform swaps MLflow champion alias → new model is live

PHASE 7 — Drift Detection
━━━━━━━━━━━━━━━
Files:   backend/platform/routers/drift.py
         backend/platform/routers/drift_trigger.py  (both exist)
Role:    Closes the fully automated loop. No manual webhooks needed after this.
         Watches prediction traffic, computes drift, alerts the agent automatically.

Logic:
  1. Every POST /predict → log prediction + features to DB
  2. Every DRIFT_WINDOW_SIZE predictions (default 500): compute PSI + chi² per feature
     PSI = sum over bins of (actual - reference) / reference  (vs Phase 5 reference stats)
     chi² = scipy.stats.chi2_contingency on categorical features
  3. If any feature PSI > DRIFT_PSI_THRESHOLD_HIGH (0.2):
       POST http://agent:8002/webhook  body: {feature_name, psi_score, chi2_pvalue, model_version}
  4. Agent handles it from there (already working)
