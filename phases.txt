PHASE 3 — Worker
Goal: Worker consumes tasks, does real work (retrain, rollback, replay), writes results back.

What we run locally:

Worker process: python -m backend.worker.main
Uses same local Postgres + Redis + MLflow
What we build:

worker/settings.py — Pydantic settings
worker/main.py — BLPOP loop, retry (1,2,4,8,16s), 5 retries → DLQ
worker/queue/dlq.py — LPUSH to dlq:tasks, write ActionLog
worker/tasks/retrain.py — train XGBoost pipeline → gate checks → MLflow Staging → notify agent for 2nd HIL
worker/tasks/rollback.py — flip MLflow alias → write DB
worker/tasks/replay_test.py — score test set → write metrics
Note on 2nd HIL: Worker calls agent POST /investigations/{id}/notify_retrain_complete → agent creates new HILApproval for Production promotion → human approves → agent calls platform /promote

Test gate:


Push manual RETRAIN task to Redis →
Worker logs: retrain_started, retrain_complete →
MLflow: version 2 in Staging →
GET /approvals → new pending approval for PROMOTE_TO_PRODUCTION →
Approve it → platform /promote called → version 2 in Production

Push task, kill Postgres after attempt 2 →
5 retries → redis-cli LLEN dlq:tasks → 1
